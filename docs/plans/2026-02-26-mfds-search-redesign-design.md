# MFDS 품목검색탭 UI/UX 재설계

> 날짜: 2026-02-26
> 접근 방식: Smart Search Bar (접근 A)

## 배경

현재 `MfdsSearchPanel` (638줄 단일 컴포넌트)의 UX 문제:

- 검색 필드가 3+4개로 나뉘어 있어 어떤 필드로 검색해야 할지 직관적이지 않음
- 의약품 24개, 의료기기 21개 컬럼이 한 번에 노출되어 테이블이 과도하게 넓음
- 검색 결과에서 원하는 품목을 빠르게 식별하기 어려움
- 품목 추가 후 동선이 단절됨

## 사용 시나리오

1. 품목명으로 빠르게 1건 찾기
2. 업체명 + 카테고리 등 조건 조합으로 여러 품목 탐색
3. 표준코드/허가번호로 정확히 1건 찾기

## 전체 레이아웃

```
┌─────────────────────────────────────────────────┐
│  품목 검색                                        │
│  식약처 API에서 의약품/의료기기를 검색합니다          │
│                                                   │
│  [의약품 | 의료기기]  ← 세그먼트 토글               │
│                                                   │
│  ┌─🔍─────────────────────────────── [검색] ─┐   │
│  │  품목명, 업체명, 코드로 검색...              │   │
│  └──────────────────────────────────────────┘   │
│  [+ 필터 추가]                                    │
│  (활성 필터 칩: [업체명: 한미약품 ✕] [등급: 2 ✕])   │
│                                                   │
│  총 1,234건  |  결과 내 검색: [____]  |  [컬럼⚙]  │
│                                                   │
│  ┌ 테이블 (핵심 컬럼 9개) ─────────────────────┐  │
│  │  품목명  업체명  코드  허가일  ...       추가 │  │
│  │  ─── 행 클릭 시 아코디언 확장 ───────────── │  │
│  │  │ 전체 상세 정보                          │  │
│  │  │                        [내 품목에 추가]  │  │
│  │  └──────────────────────────────────────── │  │
│  └──────────────────────────────────────────── │  │
│                                                   │
│  [◀] [1] [2] [3] ... [50] [▶]                     │
└─────────────────────────────────────────────────┘
```

## 검색어 자동 판별 로직

입력값 패턴에 따라 API 검색 필드를 자동 결정:

| 입력 패턴 | 의약품 검색 필드 | 의료기기 검색 필드 |
|-----------|-----------------|-------------------|
| 순수 숫자 (8자리+) | BAR_CODE, ITEM_SEQ | UDIDI_CD, PERMIT_NO |
| 숫자+하이픈 | EDI_CODE, BAR_CODE | PERMIT_NO, MDEQ_CLSF_NO |
| 영문+숫자 (A01B 등) | ATC_CODE | MDEQ_CLSF_NO |
| 한글 포함 | ITEM_NAME (우선) | PRDLST_NM (우선) |
| 영문만 | ITEM_NAME (영문명) | FOML_INFO (모델명) |

결과 0건 시 차선 필드로 1회 재시도 (최대 2회 요청).

## 필터 칩 지원 필드

| 의약품 | 의료기기 |
|--------|---------|
| 업체명 | 제조수입업체명 |
| 표준코드 | UDI-DI코드 |
| 보험코드 | 분류번호 |
| ATC코드 | 등급 |
| 허가일자 | 품목허가번호 |
| — | 모델명 |

## 기본 노출 컬럼

### 의약품 (9개)

1. 품목명 (`ITEM_NAME`)
2. 업체명 (`ENTP_NAME`)
3. 전문/일반 (`ETC_OTC_CODE`)
4. 표준코드 (`BAR_CODE`)
5. 보험코드 (`EDI_CODE`)
6. 허가일자 (`ITEM_PERMIT_DATE`)
7. 성분 (`MATERIAL_NAME`, truncate)
8. 상태 (`CANCEL_NAME`)
9. 추가 (액션 버튼)

### 의료기기 (9개)

1. 품목명 (`PRDLST_NM`)
2. 제조수입업체명 (`MNFT_IPRT_ENTP_NM`)
3. 등급 (`CLSF_NO_GRAD_CD`)
4. UDI-DI코드 (`UDIDI_CD`)
5. 품목허가번호 (`PERMIT_NO`)
6. 허가일자 (`PRMSN_YMD`)
7. 모델명 (`FOML_INFO`)
8. 일회용여부 (`DSPSBL_MDEQ_YN`)
9. 추가 (액션 버튼)

## 아코디언 확장 패널

행 클릭 시 해당 행 아래에 2컬럼 그리드로 나머지 전체 정보 표시:

- 기본 정보: 영문명, 위탁제조, 포장단위, ATC코드
- 허가/분류: 허가구분, 변경일자, 취소일자, 희귀의약품
- 상세 정보: 성상, 저장방법, 유효기간, 효능효과, 용법용량, 주의사항
- 하단: [내 품목에 추가] 버튼 또는 [✓ 추가됨] 뱃지

## 빈 상태 & 피드백

| 상태 | 표시 |
|------|------|
| 초기 (검색 전) | 안내 메시지 + 최근 검색어 칩 (localStorage, 최대 5개) |
| 검색 중 | 스켈레톤 로더 (3~4행) |
| 결과 0건 | "검색 결과가 없습니다" + 제안 메시지 |
| 검색 오류 | 토스트 에러 + 재시도 버튼 |
| 결과 내 필터 0건 | "필터 조건에 맞는 항목이 없습니다" + [필터 초기화] |

## 최근 검색어

- localStorage 키: `mfds-recent-searches`
- 최대 5개, `{ query, tab, timestamp }`
- 초기 화면에서 칩 형태 표시, 클릭 시 즉시 검색

## 페이지네이션

- 페이지 번호 직접 클릭 (최대 5개 + 말줄임)
- 현재 페이지 하이라이트

## 키보드 접근성

| 키 | 동작 |
|----|------|
| Enter | 검색 실행 |
| Escape | 아코디언 닫기 |
| ↑/↓ | 행 간 포커스 이동 |
| Space/Enter | 아코디언 토글 |

## 반응형

- 768px 이하: 테이블 → 카드 리스트 (품목명, 업체명, 코드, 상태, [+])
- 768px 이상: 테이블 유지
- 필터 칩: 모바일 가로 스크롤

## 품목 추가 동선

- 테이블 행: `+` 아이콘 버튼 (즉시 추가)
- 아코디언: `[내 품목에 추가]` 텍스트 버튼
- 추가 완료 토스트: "내 품목에 추가되었습니다." + [내 품목 보기 →] 링크
- 이미 추가된 항목: `✓ 추가됨` 뱃지

## 상태 관리

```
MfdsSearchPanel state
├── tab: "drug" | "device_std"
├── query: string                        ← 통합 검색창
├── activeFilters: FilterChip[]          ← [{field, label, value}]
├── results: Record<string, unknown>[]
├── totalCount: number
├── page: number
├── hasSearched: boolean
├── expandedRowId: string | null         ← 아코디언
├── sorting: SortingState
├── columnVisibility: VisibilityState
├── globalFilter: string
└── columnSizing: Record<string, number>
```

탭 전환 시 `query` 유지, `activeFilters`/`results`/`expandedRowId` 리셋.

## 컴포넌트 분리

```
MfdsSearchPanel (리팩토링)
├── MfdsSearchBar          ← 통합 검색창 + 세그먼트 토글
│   ├── FilterChipList     ← 활성 필터 칩
│   └── FilterAddMenu      ← [+ 필터 추가] 드롭다운
├── MfdsResultToolbar      ← 건수, 결과 내 검색, 컬럼 설정
├── MfdsResultTable        ← TanStack 테이블 (핵심 컬럼)
│   └── MfdsRowDetail      ← 아코디언 확장 패널
└── MfdsPagination         ← 페이지네이션
```

## 제거 항목

- 컬럼 드래그 재배치 (`columnOrder`, DnD)
- 고급 검색 접기/펼치기 (`showAdvanced`)
- 필드별 개별 입력 (`filters` 객체) → `query` + `activeFilters`로 대체
