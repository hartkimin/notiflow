# 대시보드 탭 + 일괄삭제 기능 설계

## 개요
- **대시보드 탭**: 홈 화면으로 메시지 현황 요약 + 빠른 접근 목록 제공
- **일괄삭제**: 기존 ViewModel의 다중 선택 기능을 UI에 연결

## 1. 대시보드 탭

### 탭 구조 변경
**Before:** 메시지 | 캘린더 | 보드 | 설정 (4개)
**After:** 대시보드 | 메시지 | 캘린더 | 보드 | 설정 (5개)

### 대시보드 화면 구성

```
┌────────────────────────────────────┐
│ 대시보드                           │
├────────────────────────────────────┤
│ ┌──────────────────────────────┐   │
│ │ 오늘의 요약    [접기/펼치기]  │   │
│ │  오늘 수신  미처리   완료     │   │
│ │    12       8       25      │   │
│ └──────────────────────────────┘   │
├────────────────────────────────────┤
│ 미처리 메시지 (8)           [더보기]│
│ ├─ 메시지 카드 (간소화)            │
│ ├─ 메시지 카드                     │
│ └─ 메시지 카드                     │
├────────────────────────────────────┤
│ 최근 메시지 (5)             [더보기]│
│ ├─ 메시지 카드                     │
│ ├─ 메시지 카드                     │
│ └─ ...                            │
└────────────────────────────────────┘
```

### 기능 상세
- **통계 카드**: 기존 `MessageStatsCard` 재사용
- **미처리 메시지**: 첫 번째 상태(미확인) 메시지 최대 5개
- **최근 메시지**: 전체 메시지 최신 순 최대 5개
- **카드 클릭**: 메시지 상세 화면으로 이동
- **더보기 클릭**: 메시지 탭으로 이동 (해당 필터 적용)

### 파일 생성
- `ui/dashboard/DashboardScreen.kt`
- `viewmodel/DashboardViewModel.kt`

## 2. 다중 선택 삭제

### 현재 구현 상태 (ViewModel)
```kotlin
// MessageViewModel.kt - 이미 구현됨
selectedIds: StateFlow<Set<Long>>
selectionMode: StateFlow<Boolean>
enterSelectionMode(id: Long)
exitSelectionMode()
toggleSelection(id: Long)
selectAll(ids: List<Long>)
bulkDelete()
bulkUpdateStatus(statusId: Long)
```

### 필요한 UI 작업

#### 선택 모드 액션바
```
┌────────────────────────────────────┐
│ [X] 3개 선택됨   [전체선택] [삭제] │
└────────────────────────────────────┘
```

#### 선택 모드 동작
- **진입**: 메시지 카드 롱프레스
- **선택/해제**: 카드 탭 (체크박스 토글)
- **전체선택**: 현재 필터된 메시지 전체 선택
- **삭제**: 확인 다이얼로그 후 bulkDelete() 호출
- **취소**: X 버튼으로 선택 모드 종료

### 파일 수정
- `ui/message/MessageListScreen.kt` - 선택 모드 UI 추가
- `ui/message/MessageCard.kt` - 체크박스 추가 (선택 모드 시)
